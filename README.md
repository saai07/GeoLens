# GeoLens — GEO Audit API


GeoLens is a Generative Engine Optimization (GEO) audit tool that analyzes any public webpage for **AI citation readiness**. It scrapes, scores, and recommends structured data improvements — helping your content get discovered and cited by AI engines like ChatGPT, Claude, and Perplexity.

---

## Overview

Modern search is shifting from links to **AI-generated answers**. When a user asks an AI engine a question, the engine synthesizes a response by citing authoritative sources. To be cited, your content must be:

- **Structured** — using semantic HTML and proper heading hierarchy.
- **Described** — with JSON-LD structured data that machines can parse.
- **Accessible** — by allowing AI crawlers in your robots.txt.
- **Rich** — with descriptive alt text, meta descriptions, and clean markup.

GeoLens audits all of this automatically and gives you a **score out of 100** with a letter grade, actionable recommendations, and a ready-to-use JSON-LD schema block generated by Google Gemini.

## Setup

### Prerequisites

- Python 3.11+
- A [Google Gemini API key](https://aistudio.google.com/app/apikey)

### Installation

```bash
# Clone the repository
git clone https://github.com/saai07/geolens.git
cd geolens

# Create a virtual environment
python -m venv venv
source venv/bin/activate   # Linux/Mac
venv\Scripts\activate      # Windows

# Install dependencies
pip install -r requirements.txt
```

### Environment Variables

Create a `.env` file or export the variable directly:

```bash
export GEMINI_API_KEY="your-gemini-api-key-here"
```

On Windows:

```powershell
$env:GEMINI_API_KEY = "your-gemini-api-key-here"
```

### Running the Server

```bash
uvicorn main:app --reload --port 8000
```

The API will be available at `http://localhost:8000`. Interactive docs at `http://localhost:8000/docs`.

## Scoring Methodology

GeoLens evaluates 5 metrics for a total of **100 points**:

| Metric | Max Points | What It Checks |
|---|---|---|
| **Schema Markup** | 25 | JSON-LD existence, valid `@type`, `name`/`description`, `url`/`image` fields |
| **Semantic HTML** | 20 | Presence of `<main>`, `<article>`, `<section>`, `<nav>` elements |
| **Image Alt Text** | 20 | Percentage of `<img>` tags with non-empty `alt` attributes |
| **Content Structure** | 20 | Single `<h1>`, ≥2 `<h2>`s, ≥1 `<h3>`, no skipped levels, no vague headings |
| **AI Bot Access** | 15 | GPTBot, ClaudeBot, PerplexityBot not blocked in `robots.txt` |

### Grade Scale

| Grade | Score Range |
|---|---|
| **A** | 80 – 100 |
| **B** | 60 – 79 |
| **C** | 40 – 59 |
| **D** | 20 – 39 |
| **F** | 0 – 19 |

Each metric returns a **status** of `pass` (≥80% of max), `partial` (≥40%), or `fail` (<40%).

The top 3 lowest-scoring metrics generate **concrete, actionable recommendations** telling you exactly what to fix.

## Where Gemini Is (and Isn't) Used

### ✅ Gemini IS used for:

- **JSON-LD Schema Generation** — Gemini analyzes the page's title, meta description, top headings, and URL to produce a context-aware, ready-to-embed JSON-LD structured data block. It selects the appropriate `@type` (Article, Organization, Product, or WebPage) and fills in realistic property values.

### ❌ Gemini is NOT used for:

- **Scoring** — All 5 metrics are computed using pure, deterministic Python rules. No LLM is involved in scoring. This ensures scores are **consistent, reproducible, and explainable**. An LLM-based score would vary between runs and offer no transparency into what was evaluated.
- **Scraping** — Page fetching and HTML parsing use httpx and BeautifulSoup respectively.
- **Recommendations** — Generated from a static mapping based on the lowest-scoring metrics.

### Why This Separation?

| Concern | Approach | Reason |
|---|---|---|
| Scoring | Rule-based | Deterministic, fast, transparent, no API cost |
| Schema Generation | Gemini LLM | Requires contextual understanding to produce realistic, page-specific structured data |
| Recommendations | Template-based | Consistent, actionable, no hallucination risk |

## How GeoLens Supports GEO & AI Citation Readiness

**Generative Engine Optimization (GEO)** is the practice of optimizing web content so that AI-powered answer engines (ChatGPT, Claude, Perplexity, Google AI Overviews) can discover, parse, and cite it accurately.

Unlike traditional SEO which targets search engine rankings, GEO targets **AI citation probability** — the likelihood that an AI engine will reference your content when generating answers.

GeoLens supports GEO by:

1. **Auditing Machine-Readability** — Checking that your content uses semantic HTML, proper heading hierarchy, and descriptive metadata that AI parsers can understand.

2. **Evaluating Structured Data** — Verifying that JSON-LD schema markup exists and contains the fields AI engines look for (`@type`, `name`, `description`, `url`, `image`).

3. **Checking Crawler Access** — Ensuring that AI-specific crawlers (GPTBot for ChatGPT, ClaudeBot for Claude, PerplexityBot for Perplexity) are not blocked by your robots.txt.

4. **Generating Ready-to-Use Schema** — Using Gemini to produce a page-specific JSON-LD block that you can embed directly, eliminating the guesswork of schema authoring.

5. **Providing Actionable Fixes** — Ranking your weakest areas and giving concrete, prioritized recommendations so you know exactly what to improve first.

The result: your content becomes **more discoverable, more parseable, and more citable** by the AI engines that are increasingly replacing traditional search.
